# 飛食 FlyEats — 外送平台經營模擬器

> 生成式人工智慧導論｜期末專題

一款以台灣外送產業為背景的**策略經營模擬遊戲**，玩家扮演「飛食」平台 CEO，在 10 個季度內與競爭對手「送香公」爭奪台北、台中、高雄三大市場，同時透過 Gemini AI 生成的動態報告，在遊戲過程中自然接觸經濟學與商業策略概念。

---

## 遊戲設計理念

傳統經濟學教學往往停留在概念層面，難以讓學習者感受到決策背後的代價與複利效應。飛食透過邊做邊學(Learning by Doing)的設計，讓玩家在面對市占競爭、現金流壓力與滿意度管理的真實取捨時，自然觸發補貼陷阱、邊際效益遞減、損失厭惡等 13 個核心概念，而非被動閱讀定義。

每局結束後，AI 顧問會根據你的實際決策軌跡生成個人化的 Series A 審查報告，並標示你這局真正體驗到了哪些概念。

---

## 核心玩法

每回合最多做出 **2 項決策**，在資金、市占、消費者滿意度與外送員滿意度之間找到平衡：

| 決策 | 效果 |
|---|---|
| 💰 補貼 | 即時搶市占，連續投入效率遞減 |
| 📣 行銷 | 下回合生效，轉化率高但有延遲 |
| 📊 抽成調整 | 調降讓利生態圈，調升觸發損失厭惡懲罰 |
| 🏗️ 區域擴張 | 永久做大市場規模，免除自然衰退 |
| 🏷️ 品牌經營 | 養滿意度、觸發品牌飛輪複利成長 |
| 🔬 科技研發 | 解鎖 AI 路由、動態定價、商家聯盟永久加成 |
| 🤝 收購對手 | 趁送香公財務脆弱時直接終結競爭 |

10 回合後達成 **4 項勝利條件中的任意 2 項**即獲勝：
- 💰 資金 ≥ 250 萬（挑戰模式 350 萬）
- 📊 市占 ≥ 60%（挑戰模式 70%）
- 😊 消費者 + 外送員滿意度各 ≥ 65（同時達標算一項）
- 💀 送香公破產或被收購

## 三種難度

| 難度 | 回合 | 機制 | 設計目標 |
|---|---|---|---|
| 🟢 入門 | 5 回合 | 4 種基本決策，關閉黑天鵝與科技樹 | 建立基本決策直覺 |
| 🟡 標準 | 10 回合 | 全機制開啟，固定強度對手 | 學習策略組合 |
| 🔴 挑戰 | 10 回合 | 動態對手反擊、消費者危機、收購機制 | 應對不確定性 |

## AI 整合

本作使用 **Google Gemini API** 在三個時機生成動態內容：

- **市場快訊**（每回合開場）：以財經新聞語氣描述本季產業動態與黑天鵝事件
- **季度經營報告**（每回合結算後）：分析決策效果，自然帶入當局觸發的經濟學概念
- **Series A 審查總結**（遊戲結束）：針對你的實際決策風格給出個人化評估，附概念學習統計

所有 AI 內容在 API 失敗時自動降級至靜態模板，不影響遊戲進行。

## 安裝與執行

```bash
# 1. 安裝套件
pip install -r requirements.txt

# 2. 設定 Gemini API Key
cp .env.example .env
# 編輯 .env，填入 GEMINI_API_KEY=你的金鑰

# 3. 啟動
streamlit run app.py
```

> 無 API Key 亦可遊玩，AI 報告將以靜態模板替代。

## 技術棧

- **前端 / 後端**：Python + Streamlit
- **狀態管理**：`st.session_state`（Phase-based 狀態機）
- **AI**：Google Gemini（gemini-2.5-flash，降級至 gemini-3.1-flash-lite）
- **平衡驗證**：純 Python 模擬器（`balance_simulator.py`），不依賴 Streamlit 或 AI
